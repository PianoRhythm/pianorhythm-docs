
`PIANORHYTHM_ENV` (values: `development`/`dev`/`localhost`, `staging`/`stg`/`stage`, `production`/`prd`/`prod`) switches host URLs and the Algolia index name between `documents_dev` and `documents`. Defaults to development.

There are no tests or linter scripts in this project.

## Content layout

Three content roots, each contributing its own URL tree:

- `docs/` â†’ site root (`routeBasePath: '/'`), so `/guides/...`, `/tutorials/...`, `/community/...`, `/troubleshoot/...` are all docs pages. Sidebars are defined in `sidebars.js` (mixes manual entries with `autogenerated` categories â€” if you add a new folder under an autogenerated category it will appear automatically; if you add a top-level section you must wire it into `sidebars.js`).
- `blog/` â†’ `/blog` (standard Docusaurus blog).
- `changelog/source/` â†’ `/changelog` via the **custom changelog plugin** (see below). Do not hand-edit files here.

URL redirects for renamed pages live in `docusaurus.config.js` under `@docusaurus/plugin-client-redirects` â€” add entries there when moving/renaming a doc.

## Custom plugins (`src/plugins/`)

- **`changelog/`** â€” wraps `@docusaurus/plugin-content-blog`. On load it reads the root-level `changelog.md`, splits on `## ` headings, rewrites `[PRFP-NNN]` â†’ YouTrack issue links and `[#NNN]` â†’ GitHub issue links, then emits one generated post per version into `changelog/source/`. A "fake hour" is assigned per entry so multiple same-day releases order correctly. **Source of truth is `changelog.md` at the repo root** â€” edit there, not in `changelog/source/` (that directory is emptied and regenerated each build).
- **`webpack/`** â€” sets dev-server CORS/COOP/COEP headers (`Cross-Origin-Embedder-Policy: require-corp`, etc.) so the docs dev server can embed/interact with the main PianoRhythm app.
- **`feature-requests/`** â€” present but currently disabled in `docusaurus.config.js` (commented out due to production CORS issues).

A small inline `docusaurus-tailwindcss` plugin wires Tailwind into postcss. Tailwind has `preflight: false` to avoid clobbering Docusaurus theme styles, and `content` is scoped to `./src/**/*.{js,jsx,ts,tsx}` only â€” Tailwind classes in `docs/` MDX will not be picked up.

## Search / Algolia

Search does **not** use the DocSearch crawler. Instead, `generate-algolia-records.js` walks `docs/`, `blog/`, and `changelog/` at build time, chunks each page's plain-text content into 300-char slices, and writes `algolia-records/records.json`. `upload-algolia-records.js` clears the target index and re-uploads in batches of 1000. This runs via `.github/workflows/algolia-index.yml` on pushes to `main` that touch content; the site reads from the index configured in `docusaurus.config.js` (`themeConfig.algolia`).

Draft pages (`draft: true` in frontmatter) are skipped by the indexer.

## Deployment

`.github/workflows/deploy-docs.yml` builds on push to `main` (when content/config files change), caches `./dist` by a sha256 of the relevant inputs, then bakes it into `docs.Dockerfile` (nginx serving `./dist`) and pushes to the DigitalOcean container registry as `pianorhythm/docs`. Because the build output is `./dist` (not Docusaurus's default `./build`), keep `--out-dir dist` in the `build` script and the Docker `COPY` path aligned.

## Versioned docs

`includeCurrentVersion: true` is set but no versions have been cut â€” all docs live under `current`. Don't introduce a `versions.json` / `versioned_docs/` setup without coordinating, since it changes the sidebar and URL scheme.
